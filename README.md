# Sort Visualizer

A real-time sorting algorithm visualizer with synchronized audio — watch and hear algorithms work their way through data, one comparison at a time.

Built with Python and Pygame. Inspired by [The Sound of Sorting](https://panthema.net/2013/sound-of-sorting/).

---

## What It Does

Each sorting operation — comparison, swap, write — is rendered as a colored bar update and mapped to a tone whose pitch corresponds to the value being touched. The result is an audiovisual representation of algorithmic behavior that makes the structural differences between algorithms immediately apparent.

You can see a merge sort's divide-and-conquer pattern unfold, hear the randomness of a quicksort pivot selection, or watch bubble sort grind through its worst-case comparisons in real time.

---

## Algorithms

| Algorithm | Time (Avg) | Time (Worst) | Space |
|---|---|---|---|
| Bubble Sort | O(n²) | O(n²) | O(1) |
| Selection Sort | O(n²) | O(n²) | O(1) |
| Insertion Sort | O(n) best | O(n²) | O(1) |
| Merge Sort | O(n log n) | O(n log n) | O(n) |
| Quick Sort | O(n log n) | O(n²) | O(log n) |

Each algorithm is implemented as a Python generator that yields operation tuples. The visualizer consumes these one frame at a time, making it trivial to add new algorithms without touching the rendering or sound layer.

## Controls

| Input | Action |
|---|---|
| `Space` | Start / Stop sorting |
| `R` | Shuffle the array |
| `Escape` | Quit |
| Mouse | All panel controls |

The control panel exposes:

- **Algorithm selector** — switch between all five algorithms
- **Array Size** slider — 16 to 512 elements
- **Speed** slider — 1 to 200 operations per frame
- **Volume** slider — 0 to 100%
- **Shuffle / Start / Stop / Sound** buttons
- **Live stats** — comparisons, array accesses, elapsed time

---

## Color Legend

| Color | Meaning |
|---|---|
| Blue | Unsorted element |
| Red | Active comparison or read |
| Yellow-Orange | Swap in progress |
| Orange | Write / set operation |
| Green | Confirmed sorted position |

## Architecture

Algorithms yield operation descriptors rather than mutating state imperatively. The visualizer steps through these at a configurable rate per frame, giving precise control over playback speed without threading.

```python
# What each algorithm yields
("compare", i, j)   # highlight indices i and j
("swap",    i, j)   # record swap, trigger tones
("set",     i, v)   # write operation
("access",  i)      # read operation
("done",)           # trigger completion sweep
```

Sound is generated by mapping array values to frequencies across a musical range and playing short sine-wave bursts through pygame's mixer. Multiple simultaneous tones are mixed on the fly.

---

## Adding an Algorithm

1. Define a generator function in `algorithms.py` that yields the operation tuples above.
2. Add it to the `ALGORITHMS` dict at the bottom of the file.

The UI, sound, and stats tracking will pick it up automatically.
